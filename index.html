在Rails中实现分布提交表单

前言

产品经理期待能够分布提交表单。第一次做这方面的，觉得有点难。

在这次，我对它的处理是提交了三次表单。

然而后来在对这个表单编辑时，又有做成一个表单。所以，理论上三次分布只提交一次表单也是可行的。

实现

介绍利用到的数据模型

Group －> Roles

一个Group中有若干的Role，而Group的创建会随之建立一个属于创建者的Role。

在实现分布提交前，已经有可以直接使用的Model了，不过我也对它有一点点细节方面的修改。

实现中会遇到的难题

在controller中的处理还是满简单的，与一次性提交的表单的区别是，将一些格式限制暂时的忽略掉。

使用Ajax做提交，与controller的配合。利用controller的render属性把新的表单传递到Ajax的data，替换掉原来的表单。

之所以要选择Ajax做处理是因为，表单输入错误的时候，给予其错误提示。

做完这部分后，发现了在第三次表单提交后，对于它的期望是跳转到另外一个页面的。然而因为remote这个属性，它一律把读取的页面都会传递给ajax做处理。

所以，在controller中第三个表单提交成功后，传递的仅仅是一个id。我在js中为他传递的内容做了判断，以此来选择是替换内容或者是跳转页面。

后续

在这件事情过去后一个月。产品经理本着精益求精的心态（喂，让我重新做这个功能。一开始是要求在第三步的时才提交表单，而不是分三次提交。后来跟经理讨论需要，把它改为第二次和第三次分别提交了两次。

对于Controller 与 view 传递数据的理解

在我们访问一个新的页面的时候，会在 controller 里面找到 action 对应的方法。它会传递全局变量给 view。 view 传递给 controller 数据只能通过 params。
但是 controller 使用 render 读取页面是没有刷新的，那么 view 被打开也不会请求相应的 action 方法。view 将继承 controller 给予的全局变量。

总而言之就是 controller 与 view 是会互相传递数据的。 controller 传给 view 的是全局变量，而 view 传递给 controller 的是 params。

如何在一个电脑上同时使用两个Git的账号

前言

又需要登录公司的账号，又想在电脑上使用自己的账号。

实现

首先是git config方面的设置，要取消掉原本对于git账号的全局设置。

再分别去不同的项目目录中，设置这个目录中项目对应的账号。

现在git方便解决了，还有ssh密钥的问题。ssh密钥生成是有带上邮箱的。所以不同的邮箱是需要不同的密钥的。

我们需要删除以前的默认名的密钥，生成新的密钥

``bash
rm ~/.ssh/id_rsa.pub
rm ~/.ssh/id_rsa
ssh-keygen -t rsa -C “your-email-address” -f “rsa_name”

#Default Git
Host defaultgit
HostName IP Address #域名也可以
User think
IdentityFile ~/.ssh/rsa_name

Go语言编程入门（二）：一些概念

什么是包？

包是结构化的一种方式。你的应用程序可以有一个包，也可以有很多包，但是你一定需要一个名为main的包。包可以由很多个以.go为后缀的文件组成。每一个以.go为后缀的文件只属于一个包。包名应该为小写字母

什么是标准库？

标准库，即Mac中pkg/darwin_amd64 目录下的文件。标准库有大量的包。

如果一个包有改动，那么它需要重新编译，引用了这个包的所用文件也需要重新编译。
－－－ 不是很理解－－－
如果 A.go 依赖 B.go，而 B.go 又依赖 C.go
编译 C.go, B.go, 然后是 A.go
为了编译 A.go, 编译器读取的是 B.o 而不是 C.o

可见性规则

当标识符（包括常量、变量、类型、函数名、结构字段等等）以一个大写字母开头则外部可用(public)，以小写开头则不可用(private)。

别名 ：引入的包可以使用别名，来避免冲突。

. 如果引用了一个包，却没有使用它，就会报错噢。
“imported and not used”

函数

main ：你必须要有一个main包，你也必须要有一个main函数。而main函数没有参数，没有返回值。

执行顺序 ：如果没有定义init()，则从第一个函数开始。

函数里的代码（函数体）使用大括号 {} 括起来。左大括号 { 必须与方法的声明放在同一行。不然，格式化的时候会变成func main() ;

注释

注释方式与javascript相同。

类型

类型可以是基本类型，如：int、float、bool、string；结构化的（复合的），如：struct、array、slice、map、channel；只描述类型的行为的，如：interface。

Go 命名规范

干净、可读的代码和简洁性是 Go 追求的主要目标。通过 gofmt 来强制实现统一的代码风格。Go 语言中对象的命名也应该是简洁且有意义的。像 Java 和 Python 中那样使用混合着大小写和下划线的冗长的名称会严重降低代码的可读性。名称不需要指出自己所属的包，因为在调用的时候会使用包名作为限定符。返回某个对象的函数或方法的名称一般都是使用名词，没有 Get… 之类的字符，如果是用于修改某个对象，则使用 SetName。有必须要的话可以使用大小写混合的方式，如 ## MixedCaps 或 mixedCaps，而不是使用下划线来分割多个名称。
